# NTUSync ‚Äî Session Handoff

**Last updated:** end of Phase 2 implementation. Read this + the two design docs before touching code.

## What this project is
NTUSync: offline-first campus transit optimizer + academic scheduler for NTU Singapore. iOS 26+, SwiftUI, **Swift 6 strict concurrency** (`SWIFT_STRICT_CONCURRENCY=complete`, `SWIFT_DEFAULT_ACTOR_ISOLATION=MainActor`). Xcode 26.6. No external package dependencies. Fully offline ‚Äî zero network calls in core flows.

Owner is a student building this as a scholarship/internship flagship. Paid Apple Developer membership arrives ~late July 2026; until then the free account hits a **10-App-ID-per-7-days limit** (needs 2 IDs: app + widget), so on-device installs may fail with "maximum App ID limit reached" ‚Äî this is an Apple account quota, not a code bug. Simulator testing is unaffected.

## Conventions that MUST be followed
- Model/value types are declared `nonisolated struct/enum` because the project defaults to `@MainActor` isolation. Follow this pattern for any new Sendable data type or it won't cross actor boundaries.
- The campus graph is an **immutable Sendable value** (`CampusGraph`), loaded from bundled JSON, never stored in SwiftData. SwiftData joins to it by `graphNodeID` STRING, never by relationship.
- Routing runs in `actor RouteEngine` (A*, time-dependent shuttle costs). No suspension points inside the search.
- ActivityKit is touched ONLY through `LiveActivityCoordinator`. `Activity` is non-Sendable with `@concurrent` methods on iOS 26 ‚Äî the gateway bridges via `Task.detached`.
- Bundled campus data (`CampusGraph.json`) is GENERATED by `Tools/generate_campus_graph.py` ‚Äî edit the Python node/edge tables and re-run, never hand-edit the JSON. Validation tests enforce geometric consistency.
- Swift Testing (`@Test`/`#expect`), not XCTest. `#expect` cannot wrap a mutating method call ‚Äî hoist the call to a `let` first.
