# NTUSync — Session Handoff

**Last updated:** end of Phase 2 implementation. Read this + the two design docs before touching code.

## What this project is
NTUSync: offline-first campus transit optimizer + academic scheduler for NTU Singapore. iOS 26+, SwiftUI, **Swift 6 strict concurrency** (`SWIFT_STRICT_CONCURRENCY=complete`, `SWIFT_DEFAULT_ACTOR_ISOLATION=MainActor`). Xcode 26.6. No external package dependencies. Fully offline — zero network calls in core flows.

Owner is a student building this as a scholarship/internship flagship. Paid Apple Developer membership arrives ~late July 2026; until then the free account hits a **10-App-ID-per-7-days limit** (needs 2 IDs: app + widget), so on-device installs may fail with "maximum App ID limit reached" — this is an Apple account quota, not a code bug. Simulator testing is unaffected.

## Conventions that MUST be followed
- Model/value types are declared `nonisolated struct/enum` because the project defaults to `@MainActor` isolation. Follow this pattern for any new Sendable data type or it won't cross actor boundaries.
- The campus graph is an **immutable Sendable value** (`CampusGraph`), loaded from bundled JSON, never stored in SwiftData. SwiftData joins to it by `graphNodeID` STRING, never by relationship.
- Routing runs in `actor RouteEngine` (A*, time-dependent shuttle costs). No suspension points inside the search.
- ActivityKit is touched ONLY through `LiveActivityCoordinator`. `Activity` is non-Sendable with `@concurrent` methods on iOS 26 — the gateway bridges via `Task.detached`.
- Bundled campus data (`CampusGraph.json`) is GENERATED by `Tools/generate_campus_graph.py` — edit the Python node/edge tables and re-run, never hand-edit the JSON. Validation tests enforce geometric consistency.
- Swift Testing (`@Test`/`#expect`), not XCTest. `#expect` cannot wrap a mutating method call — hoist the call to a `let` first.

## Features complete (core + Phase 1)
Routing (A*, 45-node/138-edge graph, 4 travel profiles), time-dependent shuttle costs, Live Activities (Lock Screen + Dynamic Island, ≤6 pushes/trip), trip autopilot (GPS-driven phase transitions), indoor dead reckoning, automatic mid-trip re-routing, crash/relaunch recovery, SwiftData schema (versioned, SchemaV1), teaching-week recurrence, settings, onboarding flow (`NTUSync/UI/Onboarding/`), course/bench management.
**Phase 1 additions:** animated route map preview (`RouteMap.swift`), full-screen live trip map (`LiveTripView.swift`), gap advisor (`GapPlanner.swift`), amenity POI layer (`Amenities.swift` + `Amenities.json`, **35 POIs**, 10 categories — expanded July 2026 from owner ground-truthing: 9 hall/cluster gyms, 7-Elevens at North Spine + Halls 4/5, Prime Supermarket at Hall 2, PEN & INC Restaurant & Bar at N2.1 North Spine Plaza), category filter chips + "Take me there" on the renamed **Explore** tab, **tappable amenity pins** (`MapPick` selection → `AmenityDetailView` sheet: category, open/closed + hours, note, Take-me-there — same trip flow as benches), gradient next-class hero card, 30 curated benches.
**Phase 2 additions (all done):**
1. **Calendar export** (`Integrations/TimetableEventPlanner.swift` pure + tested, `Integrations/CalendarExporter.swift` EventKit adapter): Settings button writes individual dated events (no RRULEs) into a dedicated "NTUSync" calendar under **write-only** access; re-export deletes and recreates the calendar (write-only can't read events). ⚠️ Calendar *removal* under write-only access is untested on a real device — if it ever fails, fall back to requesting full access.
2. **"Leave now" notifications** (`Integrations/LeaveAlertPlanner.swift` pure + tested, `Integrations/LeaveAlertScheduler.swift`): fires at `classStart − routeTime − buffer`, route computed from the user's hall per occurrence (7-day window, ≤32 pending, rolled forward each launch via `ContentView.refreshLeaveAlerts`). Scheduler is also the `UNUserNotificationCenterDelegate`; a tapped alert sets `AppEnvironment.pendingDestination`, which switches to the Route tab and prefills the planner.
3. **Look Around checkpoints** (`UI/CheckpointPreview.swift`): tap any route leg → checkpoint chips + `LookAroundPreview`; map-snapshot fallback when offline/uncovered; user-photo slot (`CheckpointPhoto` model) for indoor nodes.
4. **Bench photos** (`UI/BenchPhotoSection.swift`, shared by add + detail sheets): PhotosPicker + `UIImagePickerController` camera bridge; all photos pass through `Support/ImageProcessing.swift` (≤1600 px JPEG) into `.externalStorage`.
5. **My-hall shelf** (`Trip/HallShelfPlanner.swift` + `UI/HallShelf.swift`): hall picker in Settings (`UserSettings.homeNodeID`); Explore tab shows nearest open Eat/Groceries/Gym/Study from the hall by real walk time, plus one-tap "Route home" into the trip flow.

**Phase 2 nice-to-haves (B3, all done):**
6. **Session conflict detection** (`Integrations/SessionConflictDetector.swift` pure + tested): `ScheduleSlot` Sendable projection; `overlaps` = same weekday **AND** teaching-weeks bitmask intersection ≠ 0 **AND** half-open time-range overlap (back-to-back sessions don't clash). Surfaced live in the add-course flow (`AddCourseView` shows a non-blocking "Schedule conflict" section) and flagged with an orange triangle on the timetable list (`conflictingIndices` → `ClassSession.scheduleSlot`).
7. **`.ics` share export** (`Integrations/ICSExporter.swift` pure + tested): reuses `TimetableEventPlanner`'s dated-event expansion (one VEVENT per occurrence, no RRULEs), RFC 5545 with CRLF/UTC/TEXT-escaping/75-octet folding; shared from Settings via `ShareLink` to a temp `NTUSync-Semester.ics`. No permissions — just a file.
8. **Exam mode** (`Integrations/ExamPlanner.swift` pure + tested, `UI/ExamsView.swift`, **SchemaV3**): one-off `ExamEvent`s (date+time, duration, free-text venue, seat no.) tied to a course by code STRING (survives course deletion; venue is deliberately not a `Venue` relationship — exam halls aren't teaching venues). `ExamPlanner.phase` drives the countdown badge (upcoming/today/in-progress/finished; boundary-tested); hourglass toolbar button on Timetable opens the sheet — gradient hero card for the soonest exam, upcoming/completed sections, swipe-to-delete, minute-tick refresh.
9. **Compass mode** (`Location/CompassMath.swift` pure + tested, `UI/CompassModeView.swift`): the A4 80% alternative (true AR geo-tracking has no Singapore coverage) — a big arrow rotating toward the next route checkpoint. Pure math: great-circle bearing, relative-angle wrap-around to (-180,180], eighth-wind glyphs, and next-checkpoint selection (snap to nearest chain node, target the first leg-boundary checkpoint at/past it, advance within a 25 m arrival radius). `LocationService` gained heading updates (`startHeadingUpdates`, true-heading-with-magnetic-fallback, accuracy exposed; nonisolated delegate extracts plain doubles before the actor hop). Toggle lives on the LiveTrip header card (map ↔ compass); degrades honestly — no heading → grayed arrow + "warming up", accuracy >30° → "heading approximate" (heading is never trusted indoors, same doctrine as dead reckoning). ⚠️ Heading behaviour is untestable in the simulator — needs an on-device pass.
10. **Anti-overwhelm UX pass** (`UI/Tips/AppTips.swift`): Explore tab icon is `mappin.and.ellipse` (was `sparkles`, which read as AI — the other two sparkles uses were swapped too: gap advisor → `lightbulb`, "All places" chip → `square.grid.2x2`); Timetable toolbar collapsed 4 icons → 2 (＋ stays primary; Exams/Semester grid/Settings live in a labelled `ellipsis.circle` Menu); **TipKit** contextual tips (`Tips.configure` in `NTUSyncApp.init`), one per surface, shown once, anchored via `.popoverTip`: add-course (＋, always shows once on a fresh install — deliberately not gated on an empty timetable, so it fires even over a seeded one), route-to-class (next-class card), compass (live-trip toggle), explore (filter chips). Tips are `nonisolated` structs like all value types. During dev, reset seen-state with `Tips.resetDatastore()`.
11. **Explore power-up** (`Persistence/SchemaV4.swift`, `Support/ExploreSearch.swift` pure + tested, `UI/BenchesView.swift`): the ＋ flow is now **Add pin** — a segmented sheet adds either a study bench (original flow) or a `UserPlace` (name + any amenity category or a category-less custom pin; SchemaV4, string-key graph join, shown on the map with filter-aware visibility and its own editable detail sheet). Map gained **2D/3D pitch** and **standard/satellite** toggles (Apple-Maps-style stacked buttons; camera tracked via `onMapCameraChange`, re-pitched in place). **Campus-only search** (`.searchable` + pure `ExploreSearch.matches`: case/diacritic-insensitive, prefix-ranked, offline over graph nodes + amenities + benches + user pins — deliberately not MKLocalSearch) jumps the camera and opens the right detail sheet; buildings get a minimal `NodeDetailView`. All four detail sheets share one `startTakeMeThereTrip` helper.
